GNU C Library compatibility report

The following links provide access to a list of the function, macro and variable names which are prescribed in the Unix® specification and/or are available in the GNU C Library. Since the goal of the GNU C Library is to be cover 100% of the Unix specification empty places mark where work is needed.

The functions are not all in the libc itself. In fact we have six libraries:

  • the libc object, which contains the major part
  • the libm object, which contains the mathematics functions
  • the libdl object, which contains functions for dynamic linking
  • the libpthread object, which contains the function for thread handling
  • the libdb object, which contains the database functions
  • the libncurses object, which contains the Curses implementation

The list was produced on a Linux/GNU system and since some of the libraries or parts of the libraries is system dependent the result might not be 100% portable for all platform the GNU C Library runs on.

Since the list is too long for a single page it is splitted and each function is on a page identified by the starting character of the function name. These pages are automatically generated and are probably not checked exactly for correctness. Extracting the information from the various sources is also done automatically and so might introduce some more errors. So take all this with a grain of salt.

Warning: These pages almost exclusively consist of big tables. Some browsers still have problems with them so you might want to avoid looking at the pages. If your browser is rather slow and you just want to take a look it is now the time to reconsider downloading them.

Some more information on the tables. Normally the cell entries contain an X in case the function is available for the appropriate standard. But there are some exceptions or variants of the standard. The following can happen:

feat
it's a feature for this standard; special rules decide whether the symbol is mandatory or not
Amend1
the symbol is defined in the first Amendment to ISO C 89
C9X
the symbol is defined in the to-be-finished ISO C 9X standard
.1g
The function is desribed in the to-be-finished POSIX.1g standard for networking
Unix
The function is required for full Unix compliance.
XStd
The function is part of the extended X/Open standard and all XSI compliant systems have to provide this function.
XCur
The function is required for the extended Curses interface.

The entries in the column for the GNU C Library consist of an X and an additional extension which specified from which library this function comes from. The symbol is

c
(mainly) defined in the C library libc
m
defined in the math library libm
dl
defined in the dynamic link library libdl
db
defined in the database library libdb
cry
defined in the encryption library libcrypt
cur
defined in the curses library libncurses
pt
defined in the thread library libpthread
