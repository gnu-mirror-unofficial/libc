GNU C 라이브러리

[ English | Korean ]


다음과 같은 방법으로 GNU C 라이브러리의 버그를 보고할 수 있습니다:

  • GNU의 GNATS (GNU Problem Report Management System)에 대한 웹 기반 인터페이스. 보고된 버그들의 데이터베이스를 살펴 볼 수도 있습니다.
  • 관리자에게 버그 리포트를 보냅니다.

모든 유닉스 호환 운영 체제에는 C 라이브러리가 필요합니다: 이 라이브러리에는 "시스템 호출"과 기타 malloc 같은 기본 함수들이 정의되어 있습니다.

GNU C 라이브러리는 GNU 시스템의 공식 C 라이브러리로 사용됩니다. 이 라이브러리의 수정된 버전이 현재 대부분의 리눅스 기반 GNU 시스템에서 C 라이브러리로 사용되고 있으며, 다음 2.0 버전은 수정없이 리눅스 기반 GNU 시스템에 사용될 예정입니다.

C 라이브러리 인터페이스의 대부분을 결정해 온 것은 유닉스의 역사와 다른 여러 표준들이었습니다. 일반적으로 GNU C 라이브러리는 ISO C와 POSIX 표준을 지원합니다. 우리는 또한 표준과 충돌하지 않는 범위 내에서 대중적인 유닉스 시스템(BSD와 시스템 V 포함)의 기능들을 지원하려고 하는 중입니다.

현재 GNU C 라이브러리는 거의 다 완성되었습니다. 다른 C 라이브러리에 있는 거의 모든 유용한 함수들이 사용 가능합니다. 그렇지만 아직도 개선의 여지는 남아 있습니다. GNU C 라이브러리의 기능을 확장하거나 개선하고자 하는 분들은 아래 공개 작업란을 읽어 본 뒤 관리자들과 작업을 협의하기 바랍니다.

GNU C 라이브러리는 이식이 가능하도록 설계되었습니다. 비록 라이브러리 소스가 매우 복잡하긴 하지만, configure와 make 같은 간단한 명령으로 지원되는 모든 시스템에 대해 이 라이브러리를 만들어 낼 수 있습니다 (물론 사용 중인 플랫폼이 지원된다고 가정하고). 버전 2의 이식과 이식 상황에 대한 더 상세한 정보는 Porting the GNU C Library를 참조하십시오. GNU libc를 다른 시스템에 이식하는 일에 관심이 있는 분들은 관리자에게 연락하시기 바랍니다.

GNU libc를 개발하거나 이식하고 계신 분들은 개발자 메일링 리스트에 가입해 주십시오. 가입은 리스트 관리자에게 편지를 쓰면 됩니다. 둘러 보기만 하실 분들은 가입하지 말 것을 부탁드립니다. 새 시험판의 발표에만 관심이 있는 분들은 발표문 리스트 관리자에게 편지를 쓰십시오.

GNU C 라이브러리의 현재 상황

현재 버전은 2.0입니다. 새롭게 바뀐 앞자리 버전 번호는 주요 변화와 몇년 간에 걸친 작업을 반영하고 있습니다. 1.09.1이 80년대 스타일의 정적 라이브러리였던 반면, 새 버전 2.0은 C 라이브러리에 최신 기술을 구현하고 있습니다. 아울러 새 버전은 가장 최신 표준 (ISO C/amendment 1, POSIX.1c, XPG4.2)을 따르고 있습니다. 또 이것은 아직 완전한 국제화 인터페이스에 필요한 몇가지 함수가 빠져있긴 하지만 국제화되어 있습니다.

버전 2.0은 공유 라이브러리(사실은 여러 개의 공유 라이브러리)로 만들어집니다. 라이브러리의 몇몇 부분들은 외부로 분리되어 있으며, 필요할 때 프로세스에 동적으로 결합됩니다.

GNU libc에 관해 더 상세한 정보를 얻고자 한다면, 발표문을 읽어 주십시오. GNU libc를 GNU/리눅스 시스템 상에서 사용하시는 분들은 확장된 발표문을 읽어 주십시오.

사용 가능 여부

현재 GNU libc 버전 2.0의 시험판이 나와 있습니다. 또, 같은 디렉토리에는 GNU libc와 함께 쓸 수 있는 부가적인 패키지가 몇 가지 있습니다. 이들은 법적인 문제(암호화 소프트웨어의 수출 제한같은)나 GNU libc 개발자 그룹에 속해 있지 않은 사람들이 관리하고 있는 등의 이유로 인해 분리되어 배포됩니다.

현재 세 개의 부가 패키지가 나와 있습니다:

  • glibc-crypt에 들어 있는 DES 암호화 함수. 이 파일은 미국 외의 지역에 살고 있는 사용자가 받으면 안 됩니다. 그 대신 다음 사이트에서 받을 수 있습니다: ftp.ifi.uio.no:/pub/gnu/glibc-crypt-2.0.tar.gz
  • clone 기반 쓰레드 라이브러리. 이것은 현재 ix86, m68k, Alpha, SPARC, 그리고 MIPS 기반 GNU/리눅스 시스템에서만 사용 가능합니다. 이 라이브러리는 glibc-linuxthreads라는 이름으로 되어 있습니다.
  • GNU libc와 같이 들어있는 localedef 유틸리티의 입력으로 쓰일 로케일 정의 파일. 이 자료 파일은 glibc-localedata에 들어 있습니다.

이 부가 프로그램들의 사용법은 FAQ을 참조하기 바랍니다. 예전에 있던 NIS/YP 부가 파일은 이제 GNU libc 자체에 통합되어 있습니다.

지원되는 시스템

버전 2는 현재 GNU 시스템--허드와 리눅스 기반 GNU 시스템--만을 지원합니다. 버전 1.09.1은 더 많은 시스템을 지원했지만, 우리는 버전 2 역시 그렇게 되길 희망하고 있습니다. 이식과 이식 상황에 대한 더 자세한 정보는 Porting the GNU C Library를 참조하십시오.

공개 작업

이 목록은 가장 최신 자료가 아닙니다. 가장 최근의 변경 사항은 GNU C 라이브러리 배포판에 들어 있는 PROJECTS 파일을 참조하십시오.

  • 현재 지원되고 있지 않은 시스템으로 GNU C 라이브러리를 이식하는 일. Porting the GNU C Library를 참조하십시오.
  • GNU C 라이브러리의 메시지는 현재 독일어, 스페인어, 프랑스어, 네덜란드어, 한국어, 폴란드어로 번역되어 있습니다. 다른 언어를 쓰는 자원자들은 GNU 번역 프로젝트 관리자에게 연락하기 바랍니다.
  • POSIX와 XPG를 완벽히 지원하는 데 필요한 일부 함수들. 이들은 다음과 같습니다:
    • POSIX에 있는 wordexp.
    • XPG4.2에 있는 tgetid, waitid.
    • XPG4.2에 있는 의사 단말기 처리 (grantpt, ptsname, unlockpt).
    • XPG4.2에 따른 getdate. 이것은 현재 여러 GNU 패키지에서 사용되고 있는 것과는 다릅니다.
    • SysVr4와 XPG4.2에 따른 fmtmsg.
  • long double 수학 함수의 일부가 아직 빠져 있습니다. 이 방면에 깊은 지식을 가지고 있는 분들의 조언을 바랍니다.
  • 현재 문자열과 메모리 처리 함수의 최적화된 어셈블러 버전이 ix86, Alpha, MIPS 프로세서 용으로 나와 있습니다. 다른 아키텍쳐로의 구현을 기다립니다.
  • 더욱 정교한 알고리즘을 사용하도록 tsearch 함수를 다시 써야 합니다. 정확하게는 AVL 트리나 그에 상응하는 것을 써야 하지만 현재는 단순한 이진 트리를 사용하고 있습니다.
  • 완전한 국제화 지원을 위해 정규식 대조 함수를 확장하거나 다시 쓸 필요가 있습니다. 여기에는 문자와 콜레이션 클래스 지원이 포함됩니다. 광역 문자 버전 역시 필요합니다.

참여한 사람들

GNU libc는 원래 Roland McGrath <roland@gnu.org>가 FSF에서 일했을 때 만들었습니다. 현재의 GNU libc는 Ulrich Drepper <drepper@cygnus.com>가 관리하고 있습니다. 그밖에 다른 많은 분들이 기여했습니다.

GNU 허드에 대한 지원은 부분적으로 Thomas Bushnell, n/BSG <thomas@gnu.ai.mt.edu>와 Miles Bader <miles@gnu.org>에 의해 쓰여졌습니다. m68k 상의 GNU/리눅스로의 이식은 완료되었고 Andreas Schwab <schwab@issan.informatik.uni-dortmund.de>이 관리하고 있습니다. Alpha 용 GNU/리눅스로의 이식은 Richard Henderson <rth@tamu.edu>과 David Mosberger-Tang <davidm@azstarnet.com>이 맡고 있습니다.

GNU 홈페이지의 메인 화면으로 돌아갑니다.

자유 소프트웨어 재단과 GNU 프로젝트에 대한 질문은 gnu@gnu.org로 보내주시기 바랍니다.

GNU에 대한 질문 이외에 홈페이지 자체에 대한 질문은 webmasters@gnu.org로 보내주시고, 그밖의 연락 방법에 대해서는 연락처 안내 부분을 참고하시기 바랍니다.
